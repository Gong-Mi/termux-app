# 显存管理架构：LRU-Gen (代际缓存)

为了支持高性能、低内存占用的 Vulkan 渲染，Termux Rust 引擎引入了类似 Linux 内核 MG-LRU 的分代管理机制。

## 1. 核心背景
在移动端 UMA (统一内存架构) 下，显存溢出即为系统内存溢出。传统的 LRU 缓存在面对终端大量滚动文本时，会发生“缓存抖动”(Thrashing)，导致真正热的 UI 元素被踢出。

## 2. 实现方案
每一个存储在显存中的 `TextBlob` (文字图块) 都被包装在 `GenerationalEntry` 中：

- **current_generation**: 渲染器内部的一个全局时钟（默认每 60 帧递增）。
- **last_generation**: 记录该资源上次被访问时的“代际”。
- **分代回收**: 
  - 当缓存达到阈值 (2000 个 Blob) 时触发。
  - 仅保留最近 2 代以内的热数据。
  - 能够精准识别并删除“已滚出屏幕”的瞬时文本。

## 3. 性能优势
- **访问开销**: $O(1)$。仅需一次简单的整数赋值，无需移动双向链表。
- **内存安全**: 极大降低了 Android `LowMemoryKiller` 杀掉进程的风险。
- **视觉稳定性**: 避免了状态栏等常驻 UI 元素的重复创建带来的闪烁。

## 4. 未来扩展
该机制为 **HDR10 (10-bit)** 提供了基础设施。HDR10 的纹理更重，必须依赖此代际回收机制来维持内存平衡。
